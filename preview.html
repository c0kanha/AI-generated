1. What is an LLM?

LLM = Large Language Model.
It is a machine-learning model trained on a huge amount of text so it can predict the next token in a sequence and generate human-like language.
Large

Large

It has a very large number of learned parameters and is trained on massive text data.
Language

Language

It works with text-like sequences: words, code, symbols, sentences, and structured prompts.
Model

Model

It is not a database or a person. It is a mathematical system that maps input tokens to output tokens.

2. Input and Output

Input

What goes in?

Usually text. That can include: prompt, question, chat history, instructions, code, examples, or retrieved context.
User: Explain photosynthesis simply.
Output

What comes out?

More tokens. The model generates one token at a time until it finishes the reply.
Photosynthesis is how plants turn sunlight, water, and carbon dioxide into energy.

3. The Core Idea in One Line

An LLM does not “think” the way humans do.
At its core, it keeps predicting the most likely next token based on the tokens already given.

6. Training: How the Model Learns Before You Use It

1. Collect training text Large text datasets are used so the model sees many patterns of language, facts, styles, structures, and reasoning traces.
2. Prediction task The model is shown text and trained to predict what token should come next.
3. Compare guess vs actual token The system measures the difference between the model’s prediction and the true next token.
4. Update weights Its internal parameters are adjusted little by little so future predictions improve.
5. Repeat at huge scale Over many training steps, the model becomes much better at continuing text.

7. Inference: What Happens When You Ask a Question

Training is when the model learns.
Inference is when the model answers.
During training
Weights are updated.
During inference
Weights usually do not change. The model only uses what it already learned plus the prompt you gave it.

8. Inside the LLM: Easy Mental Model

9. Context Window

The context window is the amount of text the model can consider at one time in a single request.
What fits inside
System prompt, user prompt, previous messages, examples, retrieved documents, and current output.
If it gets too long
Some earlier text may be cut off or compressed.
Why it matters
Good answers depend heavily on having the right context in the window.

10. Does an LLM Have Memory?

Within one request Yes, it can use the tokens present in the current context window.
Permanent native memory by default No. A base LLM does not usually store new permanent knowledge from one ordinary chat turn.
Simple truth
Base LLMs are usually stateless at request time. They answer from current context plus learned weights, not from newly saved personal memories inside the model after each chat.

11. Why LLMs Sound Smart

Pattern learning
They learn many language patterns from massive text corpora.
Context use
They can use your prompt and nearby information very effectively.
Generalization
They can combine learned patterns in new ways to answer unseen prompts.

12. What LLMs Are Good At

WritingDrafting, rewriting, summarizing, explaining, translating.
Reasoning-like tasksBreaking problems into steps, comparing options, following instructions.
Code helpExplaining code, generating code, debugging patterns.
Structure conversionTurning notes into bullets, JSON, tables, plans, emails.
Style mimicryCasual, formal, concise, educational, persuasive.
Knowledge useAnswering many questions from training knowledge and provided context.

13. What LLMs Are Not Good At

Can hallucinate
They can produce wrong information that sounds confident.
No guaranteed truth
Fluent output does not mean factual accuracy.
Limited recency
Without fresh tools or live data, they may not know current events.
Context limits
They cannot consider infinite text in one go.
No human understanding
They model language patterns, not human consciousness.
Sensitive to prompt quality
Bad inputs often produce bad outputs.

14. Common Misconceptions

“It knows everything.”
No. It only knows what it learned during training plus what is in the current prompt or provided tools.
“It is searching the internet by default.”
No. Not unless the system specifically connects it to web tools or external retrieval.
“It remembers every conversation forever.”
No. A base LLM typically does not permanently learn from each ordinary request.
“It understands exactly like a human.”
No. It is extremely powerful with language patterns, but that is not the same as human understanding.

17. Final Summary

LLM in the simplest possible form:
It takes text input, turns it into tokens, processes those tokens with a transformer network, predicts the next token again and again, and produces text output. It is trained before deployment, usually does not update itself during ordinary chat, and works best when given good context and clear prompts.
