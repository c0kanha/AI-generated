How AI Converted a Scanned PDF to HTML

Every message, every actor, every direction — the complete data flow from your keyboard to the final file.

The 4 Actors

There is no single "AI." There are 4 separate actors involved, each with a different role and location:

You (Human)

Types prompts. Approves or denies tool calls. Sees final output.

Location: Your keyboard & screen

Claude Code Harness

The CLI program running on your Mac. Manages conversation, executes tools, enforces permissions.

Location: Your Mac (local process)

Anthropic API

Cloud service that receives requests and runs the LLM. Handles authentication, rate limits, billing.

Location: Anthropic's servers (internet)

Claude LLM

The language model. Reads text & images, reasons, generates text, decides which tools to call. Has no access to your machine.

Location: Anthropic's GPU cluster

The LLM is trapped in a box

The Claude LLM cannot see your screen, cannot access your files, cannot run commands, and cannot reach the internet. It can only receive text/images and produce text. Everything else is done for it by the harness running on your machine. The LLM writes "please run this bash command" and the harness decides whether to actually run it (after your approval).

The harness is the hub

Notice: the LLM never talks to you directly. The LLM never touches your files directly. Everything goes through the harness. The harness is the middleman that:

  • Sends your prompt to the API
  • Receives the LLM's response from the API
  • Executes tool calls on your Mac (with your permission)
  • Sends tool results back to the API for the LLM to read
  • Displays the LLM's final text to you

The Conversation Loop

Every interaction follows the same cycle. Here it is with all actors labeled:

Step From To What is sent How
1 You Harness Your typed prompt (text) Keyboard → terminal stdin
2 Harness Anthropic API Full JSON payload: system instructions + conversation history + your new message + list of available tools HTTPS POST to api.anthropic.com
3 Anthropic API Claude LLM The full prompt (all messages + tool definitions) Internal: loaded onto GPUs
4 Claude LLM Anthropic API Generated response: either text to display, OR one or more tool calls (name + arguments) Internal: model output tokens
5 Anthropic API Harness JSON response containing the LLM's output HTTPS response
6a Harness You If tool call: "The AI wants to run [tool]. Allow?" (permission prompt) Terminal output
6b You Harness You press Y (approve) or N (deny) Keyboard
7 Harness Your Mac Executes the tool locally (reads a file, runs bash, writes a file, etc.) Local system calls, child processes
8 Your Mac Harness Tool result (file contents, command output, error messages, image data) stdout, file I/O
9 Harness Anthropic API Tool result appended to conversation → sent as new API call HTTPS POST (another round trip)
10 LLM (via API → Harness) Reads the tool result, decides: call another tool, or respond with final text Repeat from step 3
11 Harness You If final text: displays the LLM's message in your terminal Terminal output

Every tool call = a full round trip to Anthropic's servers

When the LLM calls a tool, the result doesn't magically appear in its memory. The harness has to send the result back to the API, and the LLM processes the entire conversation again from the top (with caching optimizations). This is why 24 tool calls means roughly 24 round trips over the internet.

The Key Insight

The LLM is a brain in a jar

The LLM has no eyes, no hands, no internet connection. It sits on Anthropic's servers and receives text and images. It outputs text. That's all it can do.

But its text output can contain structured tool-call instructions like {"tool": "Bash", "command": "brew install poppler"}. The harness on your Mac parses these instructions and actually executes them.

So the magic isn't just the LLM. It's the loop:

  1. LLM thinks and outputs a tool call (text)
  2. Harness executes the tool call (action on your machine)
  3. Harness sends the result back to the LLM (text/images)
  4. LLM thinks again with new information
  5. Repeat until the LLM outputs plain text (no more tool calls) = done

This loop turned your 6-word prompt into a 476-line HTML file. Neither the LLM nor the harness could have done it alone.
